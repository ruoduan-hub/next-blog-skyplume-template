# Product

## Register

brand

## Users

Skyplume serves developers, technical writers, and independent publishers who want a fast, maintainable personal blog template. Users read long-form MDX posts, browse tags, search published writing, and customize the site without adopting a CMS.

## Product Purpose

The project provides a content-first Next.js blog template with strong SEO defaults, local search, RSS, tags, dark mode, and clean MDX authoring. Success means the template stays easy to customize, builds statically, and keeps its publishing workflow understandable from the repository.

## Brand Personality

Calm, technical, and editorial. The interface should feel precise and readable, with design decisions serving long-form content rather than decorative marketing patterns.

## Anti-references

Avoid generic SaaS landing-page treatments, heavy visual effects, decorative card grids, and dependency-heavy template abstractions that hide core blog behavior from maintainers.

## Design Principles

- Reading first: typography, spacing, and interaction should keep posts easy to scan and read.
- Own the stack: core blog behavior should live in small local modules instead of opaque template dependencies.
- Preserve behavior: infrastructure changes should not alter routing, generated content, metadata, search documents, or RSS semantics.
- Small sharp boundaries: each utility or component should have one purpose and clear imports.
- Accessible by default: interactive surfaces must remain keyboard usable and screen-reader legible.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Preserve reduced-motion compatibility, semantic HTML, keyboard navigation, visible focus states, and meaningful labels for search, subscription, and code-copy interactions.
